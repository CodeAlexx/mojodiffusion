# OT -> Mojo Port Remaining Work

Status: practical remaining-work tracker for the OneTrainer-to-Mojo port.

The port is not done when a model compiles, loads, or passes an isolated smoke.
The finish line is a OneTrainer CUDA reference and a Mojo product path that match
on byte-identical inputs for loss, gradients, updates, saved artifacts, sampler
trajectory, image output, speed, VRAM, and dtype/storage behavior.

Reference sources:

- Main OneTrainer reference: `/home/alex/OneTrainer`
- Anima reference: `/home/alex/OneTrainer-anima-ref`

Upstream OneTrainer PRs checked on 2026-06-06:

- `Nerogar/OneTrainer#1509` is open/draft. It moves the offload/checkpoint
  direction from global `gradient_checkpointing` / `layer_offload_fraction`
  fields toward per-model-part `gradient_checkpointing`,
  `activation_offloading`, `offload_fraction`, and `load_on_demand`, plus
  `release()` lifecycle semantics. The current Mojo scalar conductor policy is
  a legacy/global bridge for the checked OneTrainer checkout, not the final
  upstream parity shape.
- `Nerogar/OneTrainer#1344` is open. It expands advanced optimizers with
  scaled/spectral optimizer behavior, centered weight-decay anchors, factored
  second moment, state precision selection, SinkSGD, signed optimizer changes,
  and OrthoGrad modes. This does not change the immediate Klein LoRA AdamW
  blocker, but it adds future optimizer dispatch, save/resume, and BF16
  optimizer-state dtype parity work.

## Prioritized Next Actions

1. Build the real product runner.
   The current dry-run/dispatch layer is useful, but the missing product object
   is still the OneTrainer-style `GenericTrainer` runtime that owns start/train/end
   lifecycle, config/concepts/sample loading, model-loop selection, cache
   preflight, save/resume, sampler cadence, and failure policy.

2. Enforce resume in product loops.
   The entrypoint now validates the internal backup shape before a long run, but
   model loops still need to consume the resume state. Required proof: resume
   from saved state and continue with matching loss/update at grad accumulation
   1 and greater than 1.

3. Add per-part offload/checkpoint policy.
   Keep legacy global-field migration, but add the upstream PR #1509 shape:
   per-part transformer/text-encoder/VAE/model-part `gradient_checkpointing`,
   `activation_offloading`, `offload_fraction`, `load_on_demand`, and
   `release()` lifecycle behavior.

4. Finish dtype-boundary cleanup before claiming production parity.
   `scripts/check_train_loop_cache_contract_bindings.py --strict` now passes
   the narrow cache-boundary scan for Qwen, Ernie, Anima, Klein, Z-Image,
   Chroma, Flux.1, SD3.5, and SDXL. Klein, Chroma, Flux.1, and SD3.5 now also
   clear the fail-loud cache-derived host-F32 readiness checks where
   implemented. Z-Image is included in the same shared readiness guard and now
   clears latent/text host-F32 step-math staging, split-process sample-request
   control wiring, request contract, and strict speed/VRAM evidence; its focused
   remaining reported blocker is full-finetune product dispatch. Z-Image's
   queued request and strict-speed pass are not accepted sampled-output or speed
   parity. Every loop
   still reports broader host `Float32` carriers. Strict
   dtype guards need to pass in train/offload/sample paths before the port can
   claim full runtime dtype parity.

5. Promote samplers from contracts to product wiring.
   For each model, wire the actual OneTrainer sampler behavior into the train
   and sample paths, then measure text conditioning, denoise trajectory, VAE
   decode, image output, sampler seconds/step, and peak VRAM.
