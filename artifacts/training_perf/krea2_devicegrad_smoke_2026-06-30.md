Known limits:

- This is a synthetic fixture smoke; loss is not expected to prove convergence.
- The emitted flags include `real-cache` because the trainer uses the real cache
  reader path with `KREA2_RES_512_SYNTH=False`; the cache contents for this
  evidence file are generated by `scripts/create_krea2_devicegrad_smoke_cache.py`.
- Counters are visible accounting, not complete profiler accounting.
- The device-grad writer still performs per-block syncs to preserve streaming
  lifetime discipline. Those streaming fences are now included in `sync_count`
  as `28` per step, but hidden library/runtime synchronization is still not
  profiler-complete.
- The resident LoRA optimizer now enters the shared
  `DeviceTrainableSet`/`DeviceGradSet`/`TrainStepDeviceResult` ABI through the
  arena-backed AdamW wrapper. The higher transfer/sync counters include visible
  optimizer descriptor/scalar accounting from that shared path, not full tensor
  gradient readback.
- Full tensor readback count remains nonzero and the scorecard is labeled
  `host-grad-compat-slow`; this is not a device-fast product claim.
- Remaining acceptance gates still require ai-toolkit/OneTrainer loss replay,
  selected gradient comparison, optimizer update parity, full-surface resume
  equivalence, complete transfer/sync accounting, and a longer real-data run.
- `KREA2_TXTFUSION_LORA` is opt-in and currently blocks sampling and resume until
  those full-surface paths are gated.
