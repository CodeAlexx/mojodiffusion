## Audit findings that were NOT bugs (corrected in the ledger — do not "fix")
- **MJ-0902 / MJ-0901** — "delete the sync" would be a **use-after-free**: the sync
  guards a host staging buffer's lifetime. Only post-kernel syncs are removable.
- **MJ-1005 / MJ-1006 / MJ-1007** — fusions help **inference only**; in training the
  backward needs the intermediates (Klein saves `mlp_gate`/`mlp_up` for
  `swiglu_backward`). Landed as additive primitives.
- **MJ-0908** — ideogram "double forward" is the conductor's **gradient-checkpointing**
  recompute (`ops_record.mojo:1287`); removing it OOMs.
- **MJ-1017** — `lora_adamw_plain_fused_parity` is RED on clean HEAD but it is a
  **false-red**: the optimizer is correct (m/v match host to ~1 ulp). Two gate bugs:
  `_ulp_diff_f32` is non-monotonic for negatives (inflates worst to 4.3M), and the
  `mv_rate<1e-3` bar is unrealistic for host-CPU vs device-GPU F32 (~50% differ by 1
  ulp via FMA). Fix the *gate* (sign-magnitude ulp + worst-ulp tolerance), not the
  optimizer. Specified in the ledger.
